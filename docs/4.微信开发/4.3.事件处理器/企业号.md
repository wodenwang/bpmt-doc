# 企业号事件处理器

----------
## 基本介绍
企业号支持对以下事件、对话框和菜单进行事件处理器绑定从而实现丰富的个性化业务场景。

- 事件
	- 用户关注
	- 用户取消关注
	- 位置信息上报
	- 用户进入应用 


- 对话框消息
	- 文本
	- 图片
	- 语音
	- 视频
	- 小视频
	- 地理位置


- 菜单
	- 视图
	- 事件
		- 菜单拉取
		- 扫码推
		- 扫码弹窗
		- 拍照
		- 拍照或者照片选择
		- 照片选择
		- 地理位置选择

## 上下文

企业号事件处理器可以使用以下上下文：

| 序号 | 对象 | 说明  |
|------|-------	-|-----|
| 0 | agent | 获取当前企业号应用的函数集 agent，参考内置函数：wx.agent  |
| 1 | meida | 获取当前企业号的临时素材管理相关函数集，参考内置函数：wx.agent.media  |
| 2 | wo | 获取用户请求相关信息，比如对话框消息、各种事件等  |

wo结构描述：


| 序号 | 属性 | 说明  |
|------|-------	-|-----|
| 0 | menu | 是否是菜单事件， 比如：true说明用户点击菜单  |
| 1 | eventKey | 菜单事件的key  |
| 2 | message | 是否是对话框消息，比如：true说明是对话框的操作  |
| 3 | subscribe | 是否是用户关注事件，比如：true说明是用户关注  |
| 4 | unSubscribe | 是否是用户取消关注事件，比如：true说明是用户取消关注  |
| 5 | locationEvent | 是否是用户位置上报事件，比如：true说明用户上报位置信息  |
| 6 | text | 对话框中的文本消息  |
| 7 | image | 对话框上传的图片消息  |
| 8 | location | 通过对话框上传或者自动上报的位置消息  |
| 9 | voice | 对话框中发送的语音信息  |
| 10 | video | 对话框中发送的视频消息  |
| 11 | shortVideo | 对话框中发送的小视频消息  |
| 12 | uid | 用户的uid  |
| 13 | agentId | 企业号应用的ID  |
| 14 | agentKey | 企业号应用在BPMT中的key  |
| 15 | enterAgent | 是否是用户进入应用事件，比如：true则说明是用户进入该应用  |
| 16 | qrCode | 二维码扫码值  |

## 消息响应

- 文本消息:
	
```groovy
return ['text': '文本消息内容']
//或者保密消息
return ['text': '文本消息内容', safe: true]
```

- 图片消息： 

```groovy
return ['image': '图片的media ID']
//保密消息
return ['image': '图片的media ID', safe: true]
```
- 语音消息：
 
```groovy
return ['voice': '语音的media ID']
//保密
return ['voice': '语音的media ID', safe: true]
```
- 视频消息： 

```groovy
def video = '' //视频的media ID
def desc = '' //视频的说明
def title = '' //视频标题
return ['video': video, 'desc': desc, 'title': title]
//保密
return ['video': video, 'desc': desc, 'title': title, 'safe': true]
```

- 消息文件：

```groovy
return ['file': '文件的media ID']
//保密
return ['file': '文件的media ID', 'safe': true]
```

## 案例

- 用户进入应用欢迎词：

```groovy
//需要打开应用中“上报进入应用事件”
return [text: "您好:" + session.USER.busiName + ",欢迎使用xx应用"]
```

- 回声(文本)：

```groovy
return [text: wo.text]
```

- 回声(图片)：

```groovy
return [image: wo.image.mediaId]
```
