# 公众号事件处理器

----------
## 基本介绍

公众号支持对以下事件、对话框和菜单进行事件处理器绑定从而实现丰富的个性化业务场景。

- 事件
	- 用户关注
	- 用户取消关注
	- 位置信息上报
	- 扫码进入（旧用户） 


- 对话框消息
	- 文本
	- 图片
	- 语音
	- 视频
	- 小视频
	- 地理位置
	- 链接消息


- 菜单
	- 视图
	- 事件
		- 菜单拉取
		- 扫码推
		- 扫码弹窗
		- 拍照
		- 拍照或者照片选择
		- 照片选择
		- 地理位置选择

## 上下文

公众号事件处理器可以使用以下上下文：

| 序号 | 对象 | 说明  |
| --- | --- | --- |
| 0 | mp | 获取当前公众号函数集 mp，参考内置函数：wx  |
| 1 | meida | 获取当前公众号的临时素材管理相关函数集，参考内置函数：wx.mp('mpKey').media  |
| 2 | wo | 获取用户请求相关信息，比如对话框消息、各种事件等  |

wo结构描述：


| 序号 | 属性 | 说明  |
| --- | --- | --- |
| 0 | menu | 是否是菜单事件， 比如：true说明用户点击菜单  |
| 1 | eventKey | 菜单事件的key  |
| 2 | message | 是否是对话框消息，比如：true说明是对话框的操作  |
| 3 | subscribe | 是否是用户关注事件，比如：true说明是用户关注  |
| 4 | unSubscribe | 是否是用户取消关注事件，比如：true说明是用户取消关注  |
| 5 | locationEvent | 是否是用户位置上报事件，比如：true说明用户上报位置信息  |
| 6 | text | 对话框中的文本消息  |
| 7 | image | 对话框上传的图片消息  |
| 8 | location | 通过对话框上传或者自动上报的位置消息  |
| 9 | voice | 对话框中发送的语音信息  |
| 10 | video | 对话框中发送的视频消息  |
| 11 | shortVideo | 对话框中发送的小视频消息  |
| 12 | openId | 用户的open ID  |
| 13 | mpKey | 公众号在BPMT中的key  |
| 14 | appId | 公众号的appId  |
| 15 | sceneScan | 是否是用户再次扫码事件，比如：true则说明是老用户再次扫码进入  |
| 16 | link | 对话框中发送的链接消息  |
| 17 | scene | 带场景值的扫码进入，比如推荐用户注册的时候可以把推荐人open ID放入二维码中，用户扫码关注的时候推荐人的open ID会被带入到BPMT  |


## 消息响应
在公众号事件处理器中，groovy脚本可以遵循以下规则返回消息给用户：

- 文本消息:
	
```groovy
return ['text': '文本消息内容']
```

- 图片消息： 

```groovy
return ['image': '图片的media ID']
```
- 语音消息：

```groovy
return ['voice': '语音的media ID']
```
- 视频消息： 

```groovy
def video = '' //视频的media ID
def desc = '' //视频的说明
def title = '' //视频标题
def thumb = '' //封面图片的media ID
return ['video': video, 'desc': desc, 'title': title, 'thumb': thumb]
	```
- 音乐消息： 

```groovy
def music = '' //音乐的media ID
def desc = '' //音乐的说明
def title = '' //音乐的标题
def thumb = '' //封面图片的media ID
def musicUrl = '' //音乐链接	
def hqMusicUrl = '' //高音质链接
return ['music': music, 'desc': desc, 'title': title,
 	'thumb': thumb, 'musicUrl': musicUrl, 'hqMusicUrl': hqMusicUrl]
```
- 卡券消息： 

```groovy
return ['card': '卡券的media ID']
```
- 图文消息（点击跳转到图文消息页面）:

```groovy
return ['mpnews': '图文消息的media ID']

//或者直接media ID
return '图文消息的media ID'
```

## 案例

- 新用户关注欢迎词：

```groovy
return [text: "您好:" + session.VISITOR.nickName + ",欢迎关注创河服务号"]
```

- 回声(文本)：

```groovy
return [text: wo.text]
```

- 回声(图片)：

```groovy
return [image: wo.image.mediaId]
```
