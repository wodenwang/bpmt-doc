# 30秒快速安装

当前 `bpmt-lite v1.6.2` 推荐使用 Docker 一键启动。不需要 clone 项目，直接执行：

```bash
curl -fsSL https://github.com/wodenwang/bpmt-lite/raw/refs/tags/v1.6.2/scripts/install.sh | bash
```

一行命令会创建 `bpmt-lite/` 运行目录，下载 Docker Compose 配置、初始化脚本、升级脚本、nginx 配置模板和默认数据库，并启动服务。`v1.6.2` 起默认使用完整库 `bpmt`。

访问：

```text
http://127.0.0.1/
```

默认账号：

```text
用户名：admin
密码：admin
```

API 文档入口：

```text
http://127.0.0.1/api/docs/
```

OpenAPI 入口：

```text
http://127.0.0.1/api/openapi.json
```

如果已经有旧版本运行目录，需要升级到最新版本：

```bash
cd bpmt-lite
sh ./upgrade.sh
```

`upgrade.sh` 会拉取 BPMT Web/API 的 `latest` 镜像，按版本执行需要的数据库升级脚本，并把升级状态记录在 `.bpmt-lite/` 下。它不会覆盖当前 `docker-compose.yml`，也不会自动升级 `mariadb`、`nginx` 等第三方容器。

如果已经启动过，再替换初始化 SQL 不会自动重新导入。需要重新初始化数据库时，先确认数据已备份，再在运行目录执行：

```bash
docker compose down
rm -rf db/data
docker compose up -d
```

OAuth 第三方登录能力从 `v1.5.0` 开始可用，`v1.6.1` 进一步补齐了微信内第三方登录流程。配置和接入方式见 [OAuth 第三方登录](../11.OAuth第三方登录/README.md)。

`bpmt_min` 最小库仍可用于维护者快速验收，但不再是 `v1.6.2` 面向普通使用者的默认 Quick Start 路径。
